# Skill — sarl-module-design-3d

## Objectif

Gérer les missions 3D, Blender, design, rendu et production visuelle.

## Quand utiliser

Utiliser cette skill lorsque la demande concerne :
- veille 3D ;
- création de workflow Blender ;
- génération d’idées de scène ;
- optimisation rendu ;
- préparation assets ;
- documentation pipeline 3D ;
- scripts Blender.

## Agents recommandés

- Agent principal : designer-3d-agent.
- Recherche : research-sage.
- Documentation : docs-scribe.
- QA : qa-agent.
- Scripts Blender avancés : codex-builder.

## Entrées nécessaires

Avant de lancer la mission, vérifier :
- objectif visuel ou technique ;
- format attendu ;
- versions logicielles ;
- contraintes de licence ;
- budget rendu ;
- niveau de risque ;
- validation humaine requise.

## Workflow

1. Identifier projet, usage final et format.
2. Consulter AGENTS.md et mémoire projet.
3. Créer une tâche Kanban.
4. Choisir pipeline Blender, FreeCAD, glTF, USD ou autre.
5. Assigner designer-3d-agent.
6. Assigner codex-builder si script Blender complexe.
7. Préparer livrable ou procédure.
8. Vérifier licences, unités, échelle, fichiers et limites.
9. Demander QA ou validation humaine si livraison client.
10. Produire rapport final.
11. Écrire les apprentissages utiles en mémoire.

## Actions autorisées

- Préparer concepts.
- Proposer pipeline.
- Préparer scripts versionnés.
- Auditer assets.
- Produire documentation.
- Préparer rendus ou exports non officiels.

## Actions interdites sans validation humaine

- publication officielle ;
- livraison client ;
- suppression assets ;
- achat d’assets ;
- rendu coûteux ;
- modification pipeline production.

## Mémoire projet

Écrire en mémoire uniquement :
- procédures de pipeline validées ;
- préférences de rendu ;
- contraintes projet durables ;
- incidents ou erreurs confirmés.

Ne jamais écrire :
- secrets ;
- licences privées ;
- fichiers clients sensibles non nécessaires.
