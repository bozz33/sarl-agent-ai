# Skill — sarl-module-research

## Objectif

Gérer veille, recherche, benchmark, sources et synthèses.

## Quand utiliser

Utiliser cette skill lorsque la demande concerne :
- recherche documentaire ;
- veille ;
- benchmark ;
- comparaison ;
- synthèse ;
- rapport sourcé ;
- proposition de décision.

## Agents recommandés

- Agent principal : research-sage.
- Documentation : docs-scribe.
- Governor si sensible : sarl-governor.

## Entrées nécessaires

Avant de lancer la mission, vérifier :
- question exacte ;
- projet concerné ;
- période ou version ;
- sources attendues ;
- niveau de preuve ;
- décision attendue ;
- validation humaine requise.

## Workflow

1. Définir la question.
2. Identifier sources fiables.
3. Consulter AGENTS.md et mémoire projet.
4. Créer une tâche Kanban si recherche durable.
5. Assigner research-sage.
6. Citer sources.
7. Comparer.
8. Résumer.
9. Proposer décisions ou prochaines actions.
10. Demander validation humaine si décision stratégique.
11. Écrire mémoire si information durable.

## Actions autorisées

- Rechercher.
- Comparer.
- Résumer.
- Citer sources.
- Préparer recommandations.
- Préparer rapport.

## Actions interdites sans validation humaine

- décision stratégique ;
- achat important ;
- changement technique majeur ;
- conclusion juridique finale ;
- conclusion financière finale ;
- action irréversible.

## Mémoire projet

Écrire en mémoire uniquement :
- décisions validées ;
- sources durables ;
- préférences projet ;
- résultats de benchmark validés ;
- procédures améliorées.

Ne jamais écrire :
- mots de passe ;
- tokens ;
- clés API ;
- secrets ;
- données privées inutiles.

## Rapport final

Format obligatoire :

RAPPORT_MODULE:
PROJET:
OBJECTIF:
AGENTS_UTILISÉS:
ACTIONS_EFFECTUÉES:
SOURCES:
RÉSULTATS:
RISQUES:
VALIDATION_HUMAINE_REQUISE:
MÉMOIRE_MISE_À_JOUR:
PROCHAINE_ACTION:
