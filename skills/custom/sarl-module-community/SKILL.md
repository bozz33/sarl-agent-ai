# Skill — sarl-module-community

## Objectif

Gérer les contenus et la communication.

## Quand utiliser

Utiliser cette skill lorsque la demande concerne :
- posts réseaux sociaux ;
- calendrier éditorial ;
- réponses communauté ;
- veille tendance ;
- analyse engagement ;
- préparation de campagnes.

## Agents recommandés

- Agent principal : community-manager.
- Recherche : research-sage.
- Documentation : docs-scribe.
- QA : qa-agent.
- Governor si sensible : sarl-governor.

## Entrées nécessaires

Avant de lancer la mission, vérifier :
- projet ou marque concernée ;
- canal ;
- audience ;
- objectif ;
- ton de marque ;
- éléments interdits ;
- niveau de risque ;
- validation humaine requise.

## Workflow

1. Identifier projet, marque et canal.
2. Consulter AGENTS.md et la mémoire projet.
3. Créer une tâche Kanban.
4. Définir angle, CTA et métrique principale.
5. Assigner community-manager.
6. Demander recherche si sujet ou plateforme incertain.
7. Produire brouillon et variantes.
8. Demander QA si campagne ou contenu sensible.
9. Demander validation humaine avant publication.
10. Produire rapport final.
11. Écrire les apprentissages utiles en mémoire.

## Actions autorisées

- Préparer brouillons.
- Proposer calendrier éditorial.
- Analyser engagement.
- Préparer reporting.
- Proposer réponses non envoyées.

## Actions interdites sans validation humaine

- publication officielle ;
- annonce client ;
- communiqué sensible ;
- contenu juridique ;
- contenu financier ;
- message de crise ;
- réponse publique directe.

## Mémoire projet

Écrire en mémoire uniquement :
- préférences de marque validées ;
- règles de ton durables ;
- résultats comparables ;
- procédures éditoriales validées.

Ne jamais écrire :
- données personnelles inutiles ;
- secrets ;
- informations client non nécessaires.

## Rapport final

Format obligatoire :

ACTIONS_EFFECTUÉES:
