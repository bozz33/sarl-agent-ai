# MONTH-1-LEARNING-PLAN.md

Plan d'apprentissage 30 jours, backtest/simulation only.

## Cycle quotidien (mission bornée `daily_trading_demo`)
1. Vérifier mode backtest + kill-switch.
2. Lire éducation requise + sources.
3. Recherche / hypothèse (research note).
4. Risk review.
5. Backtest via nautilus-runner.
6. Journal + rapport quotidien.
7. Leçon en mémoire si pertinent.

## Cycle hebdomadaire (mission `weekly_trading_review`)
performance · erreurs · conditions gagnantes/perdantes · rapport hebdo ·
proposition d'amélioration · review Governor · validation humaine si règle change.

## Calendrier
- Semaine 1-2 : stabiliser le pipeline EUR/USD + EMA cross.
- Semaine 3-4 : analyse erreurs, walk-forward, sweeps bornés sur marchés et
  stratégies allow-listés, propositions d'amélioration.

## Apprentissage autorisé sans validation
journal · rapports · statistiques · propositions · tâches Kanban · mémoire non sensible.

## Apprentissage avec validation humaine
modifier une skill/règle de risque · seuil de perte · ajouter un marché ·
toute bascule vers IBKR paper ou live.

## Critères de réussite (30 j)
journal complet · 100 % backtest · 0 live · 0 violation garde-fou · rapports
présents · ≥ 50 observations exploitables · règles améliorées avec validation ·
capacité à dire NO_TRADE. On juge la discipline, pas seulement le PnL.
